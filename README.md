# V1: сайт управления задачами руководителя

Простой веб-интерфейс для постановки задач голосом, делегирования и контроля статусов.

## Пользователи и invite-ссылки

- Руководитель (Имбицалов И.И.): `index.html?invite=inv-manager-001`
- Исполнители по умолчанию:
  - Греховник А.А.: `index.html?invite=inv-grehovnik-001`
  - Забияка В.В: `index.html?invite=inv-zabiyaka-001`
  - Лошкатова С.С.: `index.html?invite=inv-loshkatova-001`
  - Тупанкова Л.Л.: `index.html?invite=inv-tupankova-001`

Если invite не передан, открывается роль руководителя.

Нового исполнителя можно добавить на панели «Быстрая постановка задачи» (только у руководителя): поле ФИО и кнопка «+»; для него создаётся свой `invite` — подсказка со ссылкой показывается под списком исполнителей и в сообщении после добавления.

## Темы и исполнители (в той же панели)

- Редактирование: рядом с полями «Тема» и «Исполнитель» кнопки **+** (добавить из поля ввода ниже) и **−** (удалить выбранный пункт в списке).
- Хранение: отдельный ключ в `localStorage` (`manager-task-v1-catalog`).
- Удаление темы запрещено, если уже есть задача с этой темой.
- Удаление исполнителя запрещено, если на него назначена текущая задача (активный исполнитель).
- Должна остаться хотя бы одна тема и хотя бы один исполнитель.

## Что реализовано в V1

- Экран руководителя "Быстрая постановка задачи":
  - Запись голосом (Web Speech API).
  - Автозаполнение текста задачи.
  - Автоподбор темы и исполнителя с ручной корректировкой.
  - Отправка задачи.
- Экран списка задач:
  - Фильтры: активные / выполненные / отмененные.
  - Разделение представления для руководителя и исполнителя.
- Экран деталей задачи:
  - Поля задачи и история событий.
  - Отмена задачи только руководителем.
  - Передача задачи другому исполнителю: выпадающий список и кнопка «Передать».
  - Изменение статуса исполнителем.
- Сообщения об ошибках и подсказки показываются в шапке (без всплывающих `alert`).

## Архитектура кода (Вариант 2)

- `app.js` — точка входа, привязка событий и рендер UI.
- `src/config.js` — константы, роли и значения по умолчанию.
- `src/storage.js` — чтение/запись `localStorage` для задач и справочников.
- `src/domain.js` — бизнес-правила (создание, делегирование, отмена, валидации, видимость).

## Ограничения и правила доступа

- Задачи видит:
  - руководитель,
  - текущий исполнитель,
  - участники цепочки делегирования через историю.
- Исполнитель не может отменить задачу.
- Отмена доступна только руководителю.
- Вся логика V1 хранится в `localStorage` браузера.

## Критерии приемки V1

- Руководитель может поставить задачу за 2 действия:
  - записать голос,
  - отправить задачу.
- После голосового ввода система автоматически выставляет текст, тему и исполнителя.
- Исполнитель может делегировать задачу другому исполнителю.
- Отмена задачи работает только для руководителя.
